# Cursor Subagents 子代理完全指南

> 官方文档：https://cursor.com/cn/docs/context/subagents

## 一句话理解

子代理就是"主 AI 派出去干活的小助手"——主 Agent 遇到复杂任务时，把部分工作分包给专门的子代理处理，处理完再汇报结果。

## 为什么需要子代理？

核心问题：**上下文窗口有限**。探索代码库、执行命令、操控浏览器这类操作会产生大量"噪声输出"，全塞进主对话会撑爆上下文。

子代理的价值：
- **上下文隔离**：中间过程的乱七八糟留在子代理里，主代理只看最终结论
- **并行执行**：多个子代理同时跑，不用排队
- **降低成本**：高消耗任务用更便宜的快速模型处理

## 三个内置子代理（无需配置，自动使用）

| 子代理 | 用途 | 为什么是子代理 |
|--------|------|----------------|
| **Explore** | 搜索并分析代码库 | 代码库探索会生成大量中间结果，导致主上下文臃肿。用更快的模型并行执行大量搜索 |
| **Bash** | 运行一系列 Shell 命令 | 命令输出通常非常冗长，隔离可以让父代理专注于决策而不是日志 |
| **Browser** | 通过 MCP 工具控制浏览器 | 浏览器交互会产生噪声较多的 DOM 快照和截图。子代理会将其过滤为更相关的结果 |

## 自定义子代理

在 `.cursor/agents/` 目录下创建一个 `.md` 文件：

```markdown
---
name: security-auditor
description: 安全审计专家。处理认证、支付相关代码时使用。
model: fast  # 可选: fast / inherit / 具体模型ID
readonly: false
is_background: false
---

你是安全专家，负责检查 XSS、注入漏洞、硬编码密钥...
```

## Skills vs Subagents 本质区别

| 维度 | Skills | Subagents |
|------|--------|-----------|
| 运行方式 | 在主 Agent 的上下文中执行 | 独立的上下文窗口，单独运行 |
| 上下文 | 共享主对话的 token | 完全隔离，不占主对话 |
| 适合任务 | 快速、单步、可复用的操作 | 耗时、多步、会产生大量中间输出 |
| 并行能力 | 没有 | 可以多个并行跑 |
| 开销 | 几乎零开销 | 有启动成本，独立消耗 token |

**用 Skills 的场景**：生成 changelog、格式化代码导入顺序、按固定模板写注释——任务简单、一次性完成、不需要单独空间。

**用 Subagent 的场景**：扫描整个代码库找安全漏洞、跑一批测试并分析失败原因、同时做"代码审查 + 更新文档 + 跑测试"三件事——任务复杂、会产生大量噪声、或需要并行。

## 实用模式推荐

- **验证代理**：任务完成后让 verifier 子代理独立验证，避免 AI 自说自话"完成了"但实际没做好
- **调试代理**：专门做根因分析，不打补丁只修根源
- **测试运行代理**：代码变更后自动跑测试

## 使用注意

- 子代理有启动开销，简单任务反而更慢
- 每个子代理独立消耗 token，5 个并行 ≈ 5 倍费用
- 不要创建几十个模糊的子代理，**2-3 个聚焦的比 50 个万能的强**
- description 写"用于通用任务"没用，**要写具体场景**
