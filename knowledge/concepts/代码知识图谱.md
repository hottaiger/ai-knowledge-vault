# 代码知识图谱

## 定义

把代码仓库解析为**结构化的图数据结构**（节点=符号/文件，边=调用/导入/继承/执行流），让 AI 编码 Agent 能像查数据库一样理解代码，而不是逐文件 grep。代表工具：GitNexus、Graphify、Understand-Anything。

核心特征：
- **本地构建**：AST 解析 + 嵌入向量（transformers.js）零 Token 跑完
- **多模态可选**：纯代码（GitNexus）vs 代码+文档+论文+图像（Graphify）
- **MCP 协议喂给 Agent**：AI 通过 `query` / `context` / `impact` 等工具实时查询

## 关键洞见

- **改代码前先查爆炸半径**——`impact` 工具揭示"改这个函数会影响谁"，比 review 提前发现风险
- **混合搜索优于纯向量**——BM25 + 语义向量双通道，能同时兼顾"精确符号名"和"自然语言意图"
- **图谱是本地产物**——`.gitnexus/` 加进 `.gitignore`，每人本地 `analyze` 构建，不入仓库
- **Worktree 共享索引有陷阱**——软链接能省事但会写覆盖，分支差异大时单独 analyze 更稳
- **精准代码 vs 语义知识是两个赛道**——GitNexus 解决"调谁/被谁调"，Graphify 解决"实现对应哪段论文"

## 工具对比

| 工具 | 定位 | 触发方式 | 索引开销 | 适用场景 |
|------|------|----------|----------|----------|
| GitNexus | AI Agent 的代码地图 | MCP 协议 | 零 Token | 改函数评估、调用链追踪 |
| Graphify | 跨模态知识编译器 | Skill 触发 | AST 零开销，语义通道有消耗 | 代码+论文+文档混合问题 |
| Understand-Anything | Claude Code 代码地图 | Skill 触发 | — | token 直降 6.8 倍 |
